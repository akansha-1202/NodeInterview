1. What is NodeJs?

Node is a open-source, cross-platform run-time environment that allows you to run JS code on server-side, built on Chrome's V8 JS engine that execute JS outside the browser.
It is asynchronous and single-threaded in nature.

2. Why NodeJs is asynchronous in nature?

Bcz it performs non-blocking operations means it doesn't block the execution of further operation.

3. What is cross-platform, single-threaded and open-source?

OPEN-SOURCE means code that is designed to be accessible publically.
CROSS-PLATFORM means NodeJS code can be run on various platforms like MAC, Windows pr where Js is installed etc.
SINGLE-THREADED means it can perform only one task at a time.

4. Blocking vs Non-blocking

Blocking ==>
It refers to the programs that blocks the execution of further operation until current operation finishes.
These methods are executed Synchronously means the program execute line by line and perform only one task at a time. The program waits until the called function returns some value, then on;y next task will execute.


Non-Blocking ==>
It refers to the program that doesn't block the execution of further operation.
These methods are executed Asynchronously means program will not execute line by line. The program doesn't wait for time-consuming task to complete and move on to the next task. The time-consuming task will execute in background while rest of the code continue to execute.

NPM stands for Node Package Manager.

Main two functions of npm are :

6. What is Module?

Module is a small unit of independent, reusable code that is desired to be used as a building block.
It can be import and export.
Types of modules are :
1. Built-in Modules ==> Modules that we can use without installation like fs, http, url, path
2. Local/User-define Modules ==> Modules that we created locally in NodeJs App.
3. Third Party Modules ==> Modules that we can use after installation like express, cors, jsonwebtoken, bcrypt etc.

Each file in Node.js is treated as a separate module, and you can define what parts of the module (variables, functions, objects, etc.) should be accessible to other modules using the module.exports or exports object.

7. What is the purpose of "module.exports" in NodeJS?

"module.exports" is a special object used to define public interface. If we want to export some variable, functions and objects from one module to another module, we use "module.exports".

8. Difference between default export and named export.

DEFAULT EXPORT ==> It allows you to export single value from a module which can be imported with any name in another module.
NAMED EXPORT ==> It allows you to export multiple values from a module and each value must be imported using its exact name inside curly in another module.

9. How do you import any module in Node.js?

When we want to import any variable, function and object fom one module to another in NodeJS, we use "require()" function.
"require()" is a in-built function that allows you to include external modules that exist in separate JS files. It basically read a Js file, execute it and then proceeds to return the export object.

11. What are the different types of HTTP requests?

The different types of HTTP requests that we use most frequently are:

  • POST ==> Used to create a new resource on the server.
  • GET ==> Used to read a resource from the server.
  • PUT/PATCH ==> Both are used to update the existing resource but the differnce is that PUT updates the whole resource and PATCH only update a part of resource specified in request, leaving rest unchanged.
  • DELETE ==> USed to delete a specified resource on the server.

CORS stands for Cross-Origin-Resource-Sharing.
CORS is a permission slip that websites give to each other to share data and resource safely. It bypass the browser's security rule called SOP(Same Origin Policy) so that websites from diffrent origin can work together easily.

17. Explain the concept of Middleware in Node.js

Middleware is a function that access request object(req) , response object(res) and access the next middleware function in req-res application cycle.
It accept three parameter i.e., request, response, next function
Middleware can cancel the request in mid of the cycle.
Types of Application :
1. Application level middleware ==> app.use("")
2. Routing Middleware ==> router.use("")
3. Error Handling Middleware

20. What are JWT tokens?

JWT stands for JSON Web Token, a secure way to transmit information between two parties in the form of a compact JSON object.
It is widely used for authentication and authorization in web applications and APIs. JWTs are stateless, secure, and efficient.
JSON Web Tokens consist of three parts separated by dots (.), which are: header, payload, and signature.
A JWT typically looks like the following : xxxxx.yyyyy.zzzzz
The header contains metadata, the payload has claims about the user, and the signature ensures integrity

21. Define Headers,Payload, Signature

  • Headers ==> The header is the first part of a JWT and contains metadata, including the token's type and the signing algorithm used.
    •           {
            "alg": "HS256",
            "typ": "JWT"
          }
  • Payload ==> The payload is the second part of a JWT and contains the user's information and additional data in JSON format.
    •           {
            "sub": "1234567890",
            "name": "John Doe",
            "iat": 1516239022
          }
  • Signature ==> The signature is the third part of a JWT and is used to verify the token's authenticity and integrity. It is generated using the header, payload, a secret key, and the specified signing algorithm.

    • The structure of a JWT looks like this: xxxxx.yyyyy.zzzzz where,
  • xxxxx is the Base64Url encoded header.
  • yyyyy is the Base64Url encoded payload.
  • zzzzz is the signature.

    • By combining the header, payload, and signature, JWTs provide a secure and compact way to transmit information between parties and are widely used for authentication and authorization in modern web applications and APIs.
